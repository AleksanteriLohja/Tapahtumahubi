# Tekoälyn käyttö – Tapahtumahubi

## Missä käytettiin
- Arkkitehtuurin suunnan varmistus (Domain/Infrastructure/App).
- EF Core -käytännöt (IDbContextFactory, migraatiot, seed).
- UI-debugin apu (kalenterin ryhmitys, komentosidonnat).
- Dokumentaation jäsentely (README, suunnitelma).

## Hyödyt
- Nopeutti suunnittelua ja vianrajausta.
- Nostatti laatua (poikkeusten käsittely, testausohjeet).
- Auttoi kurssivaatimusten mukaisen dokumentaation kokoamisessa.

## Rajoitteet
- Koodi ja ehdotukset tarkistettiin käsin.
- Ajonaikainen debuggaus tehtiin paikallisesti; lokit (Serilog) hyödynnettiin.

## Esimerkkiprompteja
- “.NET MAUI CollectionView grouped – mikä on oikea ryhmätyyppi?”  
- “Kirjoita VS Code tasks/launch, jotta F5 ajaa MAUI Windowsilla.”  
- “Laadi loppuraportin runko kurssin vaatimuksilla.”

## Yhteenveto
Tekoälyä käytettiin sallituissa puitteissa suunnittelun ja laadun tukena; vastuu tuotoksista on tiimillä.
